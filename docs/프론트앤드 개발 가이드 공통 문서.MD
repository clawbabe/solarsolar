[프론트앤드 개발 가이드 공통 문서]

(0)
기본적으로 우린 쿠키스페이스라는 회사이지만, 할인코드 정보를 제공하는 여러 개의 사이트를 운영할 예정이기 때문에 사이트 이름은 쿠키스페이스로 제작하면 안돼. 사이트명이나 메타태그, SEO, 할인 정보들은 우선 더미텍스트로 전부 채워가면서 제작해.

(1)
html, css, js로 사이트를 제작할거야. 기본적으로 백엔드 없이 프론트앤드로만 구성되어 있어야 하고, Github에 html,css,js를 올려서 사용할거야. 내가 제작하려고 하는 웹사이트는 아고다, 트립닷컴 등 여행 OTA 사이트들의 할인코드나 할인정보(프로모션 및 카드 할인 등)를 수집하여 각 플랫폼을 이용하려는 유저들에게 할인 정보를 제공하는 할인정보 사이트야. 유사한 서비스를 제공하는 사이트로는 https://www.discountspot.co.kr/을 참고하면 돼.

(2)
사이트의 전체적인 디자인은 shadcn/ui을 활용해서 버튼, 입력창, 모달, 카드 같은 기본 컴포넌트를 이용해서 통일된 디자인으로 제작해줘.

추가로, 사이트 디자인은 '우리 서비스는 이런 규칙으로 생겼다'를 색상 코드, 폰트 크기, 간격, 모서리 둥글기, 그림자 등 추후 디자인 통일성을 위해 정확한 값으로 정리해서 디자인 요소가 추가되거나 수정되거나 할 때 마다 각 사이트마다 정리를 해둬. 추후 사이트에 다른 기능이나 서비스를 추가할 때 통일된 디자인을 유지하기 위함이야.

사이트를 제작할 때 아이콘이 필요하다면 이모지를 사용하지 말고 아이콘 라이브러리를 사용해서 제작해. 아이콘은 Lucid React 라이브러리를 사용해서 전체 UI와 톤에 자연스럽게 맞게끔 사용해.

(3) 사이트 제작 전, technical-seo-prompt-v2.md 파일을 필수로 숙지 후 참고하여 테크니컬 SEO를 완벽하게 적용해줘.
